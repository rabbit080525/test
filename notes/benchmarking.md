### 1-3. 주요 UI 컴포넌트

**상품 카드**
- 이미지 (정방형/세로형 혼용) + 브랜드명 + 상품명 + 가격
- 찜 버튼 우상단 고정
- 배지: NEW / BEST / 단독 / 한정 / 세일율(%)

**개인화 추천 영역 (추천판)**
- 실시간 행동 데이터 반영 (스크롤·클릭·찜·장바구니)
- AI 추천 시나리오 50배 확대 (개편 전 대비)
- 구매 전환율 3배 향상 (공식 수치)

**디자인 시스템**
- 그레이 배경 + 화이트 카드 모듈 방식
- 탭마다 동일 컴포넌트 재사용 → 유지보수 효율화

**내비게이션**
- 상단: 검색바 + 장바구니 + 알림
- 하단: 홈 / 탐색 / 스타일 / MY
- 홈 내부: 10개 판 수평 스크롤 탭

### 1-4. 2025년 기준 핵심 인사이트

- PC 개편으로 모바일 중심 UX를 PC까지 확장하는 **옴니채널 일관성** 확보

- 객단가 23만원 유지 → 고관여 고객층 이탈 없이 프리미엄 포지션 공고화 중

## 6. 추가 조사 결과 (2026-04-10 업데이트)

### ✅ 에이블리 AI 콘텐츠 탭 → 상품 구매 전환 경로

**전환 흐름 (3단계)**
```
[콘텐츠 탭 진입]
  → AI 옷입기 / AI 프로필 / AI 채팅 / 운세 체험
        ↓ (콘텐츠 소비 후 자동 연동)
  → 해당 콘셉트·스타일 어울리는 상품 추천 피드 노출
        ↓
  → 상품 카드 탭 → 상세 페이지 → 구매
```

**전환 메커니즘 상세**
- **AI 프로필(가상피팅)**: 사용자가 스타일 선택 후 사진 업로드 → AI 생성 이미지 하단에 착용 상품 정보 + 바로 구매 버튼 노출
- **쇼핑몰 전용 AI 프로필**: 입점 쇼핑몰 모델과 동일 착장 이미지 → 착용 상품 직접 구매 연결
- **AI 채팅**: 퍼스널 쇼퍼처럼 취향 대화 → 상품 추천 링크 제공
- **AI 프로필 찍은 유저**: 해당 콘셉트 어울리는 스타일 아이템이 홈 피드에 자연스럽게 노출

**확인된 성과 수치**
- 영앤리치 쇼핑몰 AI 프로필 적용 후 모델 착장 상품 판매량 전월 대비 **+20%**
- AI 프로필 서비스 기간(3월 25–29일) 전체 거래액 직전 동기 대비 **+45%**
- AI 스타일 서비스 매출 론칭 후 2개월 대비 **5배(+418%)** 증가 (2025.11~12 기준)
- 2025년 플랫폼 전체: 거래액 **2.5조 원**, 영업이익 **130억 원**, MAU 최초 **1,000만** 돌파(5월)

### ✅ 무신사 추천판 AI 알고리즘 구체적 모델

공개된 AWS 기술 블로그 기준으로 **스냅(Snap) 이미지 기반 추천 시스템** 구조가 확인됨.
홈 피드 추천판의 협업 필터링 상세 모델은 미공개 → 아래 구조 중 스냅 관련 부분만 1차 자료 기반.

**확인된 기술 스택 (스냅 이미지 추천)**

```
[사용자 업로드 이미지 / 스냅 피드]
  → AWS Lambda + Amazon S3 (이미지 저장·수신)
        ↓
  → YOLOS-Fashionpedia (객체 검출)
    - 트랜스포머 기반 패션 아이템 인식
    - 다양한 각도·복잡한 배경에서도 고성능
        ↓
  → FashionCLIP 2.0 (임베딩)
    - OpenAI CLIP을 패션 데이터셋으로 파인튜닝
    - 이미지 + 텍스트를 동일 벡터 공간에 임베딩
    - 스타일·카테고리·색상·소재 등 패션 특유 속성 반영
        ↓
  → OpenSearch (벡터 유사도 검색)
    - 임베딩 벡터 저장 및 유사 상품 검색
        ↓
  → 유사 상품 추천 결과 반환
```

**텍스트 검색 처리 구조**
```
자연어 검색 질의
  → Amazon API Gateway → AWS Lambda
  → AWS Step Functions (오케스트레이션)
  → Amazon Bedrock: Nova Lite (쿼리 패션 도메인 변환)
  → Amazon Bedrock: Titan Text Embedding v2 (벡터 변환)
  → OpenSearch 벡터 검색 → 결과 반환
```

**홈 피드 추천판 알고리즘 (추정)**
- AI 추천 시나리오 50배 확대 발표 기준, **협업 필터링 + 행동 시퀀스 기반 딥러닝** 하이브리드로 추정

### ⚠️ 29CM 발견 탭 취향 스와이프 → 추천 반영 속도

**확인된 사실**
- 발견 탭 내 취향 스와이프 기능 존재: 상품 스와이프 → 취향 데이터 수집 + 마일리지 적립
- "취향도, 마일리지도 쌓을 수 있는 취향 스와이프" 소개 문구 확인

**미확인 사항 (공개 자료 없음)**
- 스와이프 데이터가 피드 추천에 반영되는 **시점** (즉시 반영 vs 배치 처리)
- 반영 범위: 발견 탭 한정인지 전체 홈 피드에도 영향을 주는지 불명확
- 추천 모델과의 연동 구조 미공개

**추정**
- 마일리지 적립 게임화 구조로 보아 데이터 수집이 1차 목적
- 실시간 피드 반영보다 배치 학습 후 다음 세션에 반영하는 방식일 가능성이 높음
- 29CM는 MD 큐레이션 중심이므로 스와이프 데이터가 알고리즘보다 **MD 편집 보조** 도구로 활용될 가능성

> **결론: 공개 자료로 작동 방식 확인 불가. 추정에 의존. 직접 앱 테스트 또는 29CM 공식 채널 확인 필요.**
