# 개인화 피드 페이지 UX/UI 패턴

> 작성일: 2026-04-10

---

## 1. 피드 레이아웃 유형

### 1-1. 무한 스크롤(Infinite Scroll) 피드
- 페이지 하단 도달 시 자동으로 콘텐츠 로드
- 탐색 세션 길이 증가, 상품 노출 기회 극대화
- **필수 보완 요소**: 스티키 Back-to-Top 버튼, 필터/정렬 고정 바

### 1-2. 카드 그리드(Card Grid)
- 2열(모바일) / 3–4열(데스크탑) 상품 카드 배열
- Pinterest 스타일의 **Masonry 그리드** (비율이 다른 카드 혼합)
- 일관된 카드 크기 vs. 가변 카드 — 버티컬 성격에 따라 선택

### 1-3. 혼합 피드(Mixed Feed)
- 상품 카드 + 콘텐츠 카드(룩북, 비디오, 매거진) 교차 배치
- 패션·뷰티 버티컬에서 효과적 (콘텐츠로 브랜드 톤 유지)
- 예: 상품 3개 → 스타일링 영상 1개 → 상품 3개

## 2. 핵심 UI 컴포넌트

### 2-1. 상품 카드(Product Card)
```
┌────────────────┐
│  [상품 이미지]  │ ← 고화질, 호버 시 두 번째 이미지 전환
│   위시리스트 ♡  │ ← 우상단 고정
├────────────────┤
│ 브랜드명        │
│ 상품명          │
│ ₩39,000        │ ← 할인율 표시 (취소선 + 할인가)
│ ★4.8 (1.2k)   │ ← 별점 + 리뷰 수
│ [장바구니 담기] │ ← 카드 하단 스티키 또는 호버 시 노출
└────────────────┘
```

**추가 배지 유형**
- NEW / BEST / 한정수량 / 오늘마감 / 재입고

### 2-2. 피드 상단 내비게이션
- **카테고리 칩(Chip) 필터**: 수평 스크롤 가능, 선택 시 피드 즉시 필터링
- **개인화 태그 표시**: "내 취향", "최근 본 브랜드", "자주 산 카테고리"
- **정렬 드롭다운**: 인기순 / 최신순 / 낮은가격순 / 추천순

### 2-3. 섹션 구분 모듈
| 섹션 | 설명 |
|------|------|
| 오늘의 추천 | 실시간 AI 개인화 추천 |
| 최근 본 상품 | 세션/로컬 이력 기반 |
| 이 브랜드도 좋아할 것 같아요 | 구매 브랜드 유사 브랜드 |
| 지금 인기 급상승 | 실시간 트렌딩 |
| 재입고 알림 상품 | 관심 상품 재고 복귀 |
| 한정 타임딜 | 카운트다운 + 잔여 수량 |

### 2-4. 스티키 요소
- **하단 내비게이션 바** (모바일): 홈 / 피드 / 검색 / 장바구니 / 마이페이지
- **스티키 "바로 구매" / "장바구니"** 버튼: PDP 진입 없이 빠른 구매 유도
- **상단 검색바**: 스크롤 내려도 검색 접근성 유지

## 3. 개인화 피드 진입 시나리오별 UX

### 신규 사용자 (콜드 스타트)
1. 온보딩에서 관심 카테고리·브랜드 선택 (3–5개)
2. 선택 기반 초기 피드 구성
3. 첫 탐색 행동 즉시 반영해 빠르게 피드 개선

### 재방문 사용자
1. 마지막 세션 이후 신상품 우선 노출
2. 위시리스트 상품 가격 인하 알림 카드 삽입
3. 최근 조회 카테고리 위주 피드 재구성

### 구매 완료 후
1. 구매 상품과 보완 상품 추천 (크로스셀)
2. 동일 브랜드의 다른 상품 추천
3. 다음 구매 사이클 예측 (소모품의 경우 재구매 유도)

## 4. 인터랙션 & 애니메이션 패턴

- **로딩 스켈레톤(Skeleton UI)**: 카드 로드 전 회색 플레이스홀더로 레이아웃 안정성 유지
- **페이드인 전환**: 새 카드 로드 시 부드러운 등장 애니메이션
- **스와이프 액션**: 카드 좌우 스와이프로 "관심 없음" / "위시리스트 추가"
- **탭 즉시 반응**: 좋아요·위시리스트 탭 시 즉시 시각 피드백 (낙관적 UI 업데이트)
- **Pull-to-Refresh**: 당겨서 새로고침으로 피드 갱신

---

## 5. 접근성 & 성능 고려사항

- 이미지 Lazy Loading + WebP 포맷으로 로딩 속도 최적화
- 무한 스크롤 시 가상화(Virtualization)로 DOM 노드 수 제어
- 스크린 리더를 위한 상품 카드 aria-label 제공
- Google PageSpeed Insights 기준 Core Web Vitals 최적화 필수
- **LCP(Largest Contentful Paint)** 목표: 2.5초 이하
